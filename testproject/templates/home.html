Welcome to django Dynamic Logging test page

this project hosted on github is meant to update the python logging setting at runtime and in live running systems.
to try it in this demo, go to the admin , login and play with the configurations.

how to start

create a config

a config represent the full logging configuration for all the python process that run django. only one config can be enabled at a given time. a config can override the folowing logging property:

  • handlers level
  • loggers list (remove,add)
  • logger level
  • logger handlers list
  • logger propagate

create a trigger

to enable a config, one of his triggers must be enabled and the latest valid trigger. a Trigger have a start_date and a end_date. if the current time is between them, your trigger is valid. if you have more than one trigger valid at a given time, the one with the latest start_date will be the chosen one. you can put an empty date into start_date and end_date which mean from startup and forever respectively.

check the current config

to check the current config, the admin's config list will tell you which one is active.
